# Data Collector Hub - 总览文档

> 插件化数据枢纽 —— 统一接入多源数据，为下游应用提供标准化数据服务

---

## 1. 产品愿景与定位

**Data Collector Hub** 是一个**插件化数据枢纽**，定位于数据底座，为各类下游应用提供统一的数据接入、存储与服务能力。

**核心设计原则**：
- **DCP 只是其中一个上游采集平台**，通过 SourceEvent 标准化接入
- **数字沙盘只是其中一个下游应用**，通过 Sandbox API 消费数据
- 系统保持开放，支持任意上游数据源和下游消费方

## 2. MVP边界

### MVP核心目标

接入一个数据源（插件或上游系统），能够定时采集或通过 Ingestion API 接收数据，并通过 REST API / Sandbox API 提供给下游工具。

### MVP功能边界（摘要）

| 包含 | 不包含 |
|------|--------|
| 1个默认插件 + 定时调度 | 进程级隔离 |
| REST API + RSS/WebSocket + MCP | 多进程/分布式并发 |
| SourceEvent 接入（Ingestion API） | 事件驱动实时流 |
| 归一化处理（Normalizer） | 用户管理和权限控制 |
| Sandbox API（下游应用接口） | 复杂数据加工 |
| Web管理界面（只读） | |
| 插件监控告警 | |

> 📄 **详细边界定义**：[02-prd.md §4. MVP范围](./02-prd.md#4-mvp范围)

### 成功指标（摘要）

| 类别 | 关键指标 |
|------|----------|
| 可靠性 | 采集成功率 ≥ 99%，调度准时率 ≥ 99.9% |
| 可用性 | 单插件失败不影响其他任务 |
| 性能 | API P99 < 200ms，WebSocket < 1s |
| 扩展性 | 协程级并发 ≥ 2 |

> 📄 **完整SLA定义**：[02-prd.md §5. SLA / 成功指标](./02-prd.md#5-sla--成功指标)

## 6. 关键设计原则

1. **插件独立性**：插件之间不允许有依赖，每个插件必须是独立完整的数据采集工具
2. **上游开放**：DCP 只是其中一个上游，通过 SourceEvent 标准化接入任意上游系统
3. **下游开放**：数字沙盘只是其中一个下游，通过 Sandbox API 服务任意下游应用
4. **逻辑隔离**：配置独立、错误边界独立（非进程级强隔离）
5. **延迟加载**：PluginManager AST解析元数据，用时才import
6. **受控执行**：Semaphore控制并发 + asyncio.wait_for控制超时
7. **单轮询广播**：WebSocket采用单轮询广播模式（详见ADR-003）
8. **多层数据**：raw（原始）→ normalized（规范化）→ canonical（实体）
